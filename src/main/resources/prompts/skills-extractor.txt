You are an expert recruiter extracting skills from a CV for automated job matching.

Use ONLY evidence explicitly stated or clearly demonstrated in the CV.
Do NOT infer or assume unstated skills.

CV TEXT:
%s

TASK
Extract skills into the following categories:

1. technicalSkills
Technologies, tools, software, platforms, frameworks, or technical methods explicitly mentioned
(e.g. Java, AWS, Docker, SQL, Excel, Jenkins)

2. professionalSkills
Job-related competencies, methodologies, domain expertise, certifications, or practices
(e.g. Project management, Stakeholder engagement, Business analysis, PRINCE2)

3. softSkills
Interpersonal or behavioral skills ONLY if clearly evidenced by actions
(e.g. Leadership → “managed team”, Communication → “presented reports”)

4. qualifications
Formal degrees, diplomas, certifications, or accredited training
(e.g. BSc Computer Science, PRINCE2 Practitioner)

RULES
- Extract exactly as written or clearly evidenced.
- Preserve original wording where possible.
- Do not include job titles, employers, dates, or vague adjectives.
- Do not repeat the same skill across categories.
- Categories may be empty if no evidence exists.

OUTPUT
Return ONLY valid JSON in this structure:

{
  "technicalSkills": [],
  "professionalSkills": [],
  "softSkills": [],
  "qualifications": []
}
